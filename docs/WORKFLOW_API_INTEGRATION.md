# Workflow API Integration Guide

This implementation adds workflow status API endpoints to the Alcove Bridge.

## Files Added

- internal/bridge/workflow_api.go - Type definitions for workflow API responses
- internal/bridge/workflow_api_handlers.go - HTTP handlers for workflow endpoints

## Integration Required

Add these lines to cmd/bridge/main.go:

After api := bridge.NewAPI(...):
workflowHandlers := bridge.NewWorkflowAPIHandlers(api, workflowStore, workflowEngine)

After api.RegisterRoutes(mux):
workflowHandlers.RegisterWorkflowRoutes(mux)

## API Endpoints Implemented

GET /api/v1/workflows - List workflow definitions
GET /api/v1/workflows/{id} - Get workflow definition detail  
GET /api/v1/workflow-runs - List workflow runs with status filter
GET /api/v1/workflow-runs/{id} - Get run detail with all steps, outputs, status
POST /api/v1/workflow-runs - Manually trigger a workflow run
DELETE /api/v1/workflow-runs/{id} - Cancel a running workflow

## Token/Cost Tracking

Per-step token counting includes:
- tokens_in - Input tokens consumed by the step
- tokens_out - Output tokens generated by the step
- duration_seconds - Step execution duration

Extracted from session proxy logs for cost attribution and debugging runaway agents.
